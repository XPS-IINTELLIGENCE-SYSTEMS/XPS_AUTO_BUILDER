# SOURCE_OF_TRUTH_MANIFEST

## Source priority
1. Explicit user instructions in the current conversation
2. Connected target repo state that is readable and writable in runtime
3. Connected target Drive folder state that is readable in runtime
4. Uploaded working artifacts in the current conversation
5. Verified XPS canon files and manifests
6. Verified connector reality from available tools
7. Prior drafts, mirrors, and quarantine copies

## Classification law
Every serious pass must classify findings as one of:
- `verified`
- `inferred`
- `missing`
- `blocked`
- `quarantined`

## Builder truth model
- Repo state is implementation truth for this build pass.
- Drive state is canon/reference truth when present.
- Scheduler state is recurrence truth only when an actual task is created.
- Skill library state is orchestration truth for what specialized agents can be invoked.

## Promotion law
An artifact becomes active canon only after:
1. it is generated
2. it is validated against naming and architecture rules
3. its destination is verified
4. touchdown is re-read
5. the ledger is updated
