# UNIVERSAL_BUILD_SYSTEM_SPEC

## Purpose
`XPS_AUTO_BUILDER` is a bounded universal builder. It can design, assemble, repair, package, and schedule **any target system** using verified canon, visible skills, and approved destinations.

## Supported operating modes
1. `initialize` — create the first artifact pack for a new target system
2. `rehydrate` — read current state and resume from the latest valid point
3. `expand` — add missing modules, prompts, contracts, or schedules
4. `repair` — fix bounded defects, drift, duplicates, or broken handoffs
5. `validate` — score completeness, coherence, and deployability
6. `package` — produce a release-ready builder pack
7. `handoff` — produce a scheduler-ready or operator-ready run surface

## Target-system contract
Every target build must define:
- target system name
- mission
- primary surfaces
- source-of-truth order
- required components
- allowed write paths
- scheduler cadence
- validation gates
- release criteria

## Build sequence
1. rehydrate from accessible canon
2. audit connector reality
3. build or refresh the target profile
4. select the next missing or weakest artifact
5. route to the correct skill set
6. generate bounded artifacts
7. validate naming, architecture, and destination safety
8. publish to verified surfaces
9. update ledger, backlog, and next recursive actions

## Non-negotiables
- never rename canonical XPS systems
- never claim a live runtime that is not proven in the current runtime
- never treat inferred destination access as verified write access
- never perform destructive cleanup without rollback notes
- never expand into scheduler sprawl when one primary loop is sufficient

## Success condition
The builder succeeds when a target system can be rehydrated, extended, and maintained by a recurring bounded loop using repo canon, scheduler prompts, and explicit skill routing.
